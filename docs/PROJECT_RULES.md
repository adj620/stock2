# 프로젝트 통합 규칙 (Stock2)

본 프로젝트는 주식 잔고 및 매매 데이터를 관리하고 분석하는 시스템으로, **구글 스프레드시트를 단일 데이터 소스**로 사용합니다.

## 1. 프로젝트 구조
```text
Stock2/
├── run_update.bat (주 실행 파일)
├── src/
│   └── update_holdings.py (주 로직)
├── data/
│   ├── sector_mapping.json (코드별 섹터 매핑)
│   └── sector_mapping_by_name.json (이름별 섹터 매핑)
├── config/
│   └── stock-holding-log-db46e6d87dd6.json (구글 클라우드 서비스 계정 인증키)
└── docs/
    └── PROJECT_RULES.md (본 문서)
```

## 2. 데이터 관리 원칙
- **Single Source of Truth**: 모든 데이터(전일 잔고, 매매일지, 오늘 잔고)는 구글 스프레드시트에서 관리합니다. 로컬 엑셀 파일은 사용하지 않습니다.
- **인증**: `config/` 폴더의 서비스 계정 키를 통해 구글 API에 접근합니다.
- **날짜 계산**: 영업일(주말/휴일 제외) 기준으로 전일 영업일을 자동 계산하여 데이터를 매칭합니다.

## 3. 섹터 정보 관리
- **자동 페치**: 매핑 파일에 없는 종목은 네이버 증권에서 자동으로 업종(섹터) 정보를 가져와 `data/` 폴더의 JSON 파일에 영구 저장합니다.
- **매칭 순서**: (1) 코드 매핑 확인 -> (2) 이름 매핑 확인 -> (3) 네이버 페치

## 4. 계산 및 서식 규칙
### 4.1. 핵심 계산 (대체식 금지)
- **평가금액**: `현재가 × 잔고수량`
- **매입금액**: `평균단가 × 잔고수량` (주의: `평가금액 - 미실현손익` 방식 금지)
- **미실현손익**: `평가금액 - 매입금액`
- **손익률(%)**: `((현재가 / 평균단가) - 1.0) × 100.0` (평균단가=0이면 0% 처리)
- **보유비중(%)**: `(해당 종목 평가금액 / 전체 평가금액) × 100.0`

### 4.2. 구글 시트 서식 규칙
- **색상**: 손익률 > 0이면 행 전체 빨강, 손익률 < 0이면 행 전체 파랑 적용
- **음영**: 전일 대비 수량 차이가 있는 행은 배경색 `#F0F0F0` 적용
- **서식**:
    - 숫자: 천 단위 콤마 (`#,##0`)
    - 퍼센트: 손익률 소수점 2자리 (`0.00%`), 보유비중/비중차이 소수점 9자리 (`0.000000000%`)
- **기타**: 첫 행 틀 고정, 열 너비 자동 맞춤(또는 지정된 픽셀값 적용)

## 5. 실행 및 디버그
### 5.1. 표준 실행 (run_update.bat)
```bash
python src/update_holdings.py --date YYYYMMDD --override-price "종목명=가격" ...
```
### 5.2. 옵션 가이드
- `--override-price`: K-OTC 등 가격 조회가 안 되는 종목의 가격을 수동으로 입력할 때 사용합니다.
- `--fetch-prices`: FDR을 통해 실시간 가격을 업데이트합니다.
- `--dry-run`: 실제 시트에는 쓰지 않고 콘솔 창에 결과만 출력합니다.

## 6. 주의 사항
- PowerShell에서 실행 시 한글 깨짐 방지를 위해 `chcp 65001` 명령어를 먼저 실행해야 합니다.
- `잔고수량 > 0` 인데 `평균단가 = 0` 인 행이 발생하면 계산 오류를 막기 위해 프로세스가 중지됩니다.
